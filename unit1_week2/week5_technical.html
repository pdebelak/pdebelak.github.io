Fun with Ruby Classes

Classes are an important part of Ruby. What is a class and how should it be used? Let's say you have a cat. A cat is not a class. But your cat is an instance of the class Cat. That's a good example. Let's take a look.

Creating a new class

The least you need to do to create a new class is this:

  class Cat
  end

Looks great! But it doesn't really do anything yet.

Initializing initialization sequence

To add a little bit more to the class, let's set up what happens right away when an instance of the class is created. We do that with an initialize method.

  class Cat
    def initialize(name, color, age)
      @name = name
      @color = color
      @age = age
    end
  end

Now when we create a new cat, we can assign the name, color, and age of that cat. Wondering what is with those @ symbols? Those mean that @name and the like are instance variables. They are different for each instance of that class, but each instance has those variables.

Other methods

Maybe we want the cat to be able to do some things. That seems appropriate. Every good cat can cuddle and shed. Let's make sure our cats can do this.

  class Cat
    def initialize(name, color, age)
      @name = name
      @color = color
      @age = age
    end
    def cuddle
      "#{@name} cuddles with all the cuddling knowledge learned in #{@age} years."
    end
    def shed
      "#{@name} gets #{@color} hair over everything you own."
    end
  end

I think that about covers all the important tasks a cat must perform.

Putting it all together

Okay! Now let's make a couple instances of the Cat class. We'll use my cats as examples.

  ash = Cat.new("Ash", "gray and white", 9)
  mabbus = Cat.new("Mabbus", "black", 19)

Now that they are created, we can call the above methods!

  ash.cuddle # returns "Ash cuddles with all the cuddling knowledge learned in 9 years."
  ash.shed # returns "Ash gets gray and white hair over everything you own."
  mabbus.cuddle # returns "Mabbus cuddles with all the cuddling knowledge learned in 19 years."
  mabbus.shed # returns "Mabbus gets black hair over everything you own."

Makes sense? Great!

Classes are an important way in Ruby to avoid a lot of unnecessary repetition in your code. If you find that you have a bunch of objects that need to have similar attributes and perform similar tasks (methods) then think about creating a class for those objects.
