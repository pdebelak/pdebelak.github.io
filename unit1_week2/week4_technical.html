Mapping it out - the map method in Ruby

Sometimes you want to do the same thing to every item in an array or hash. As always in Ruby, there is more than one way to accomplish this. If you want your outcome to be an array, though, it is likely that map is your best bet.

How Map works

Let's say we are working with an array called hats:

  hats = ["bowler", "top", "baseball"]

I'm already excited! If you are excited, too, maybe you want to indicate this by adding an exclamation point to each item. You could do it like this:

  excited_hats = []
  hats.each do |hat|
    excited_hats << hat + "!"
  end
  # output
  excited_hats = ["bowler!", "top!", "baseball!"]

That's all well and good, but there is a better way. Let's do that same thing with map:

  hats.map { |hat| hat + "!" }
  # output
  ["bowler!", "top!", "baseball!"]

That's much better!

Hey! How about hashes?

I'm glad you asked. With hashes, the output is an array, just like when you map on an array. Let's see an example:

  best_hash_ever = { "a"=>1, "b"=>2, "c"=>3, "d"=>4 }
  best_hash_ever.map { |letter, number| letter * number }
  # output
  ["a", "bb", "ccc", "dddd"]

Makes sense? It looks so clean and clear!

What's the deal with collect?

You may occasionally see the .collect method in Ruby and it looks suspiciously like .map. What's the deal? The two are actually identical. I said that there is more than one way to do things in Ruby, and I really wasn't joking.

When you want to do something to each item in an array or hash and you want your output to be an array, map is the perfect method. Next time you are doing .each, quickly think if .map might be a bit better.
